Guides & Reference

Documentation and guides for working with OT protocols and security tools.

Modbus Guide

Learn Modbus TCP frame structure, function codes (01-23), and common attack vectors.

  • Function Code 1: Read Coils
  • Function Code 3: Read Holding Registers
  • Function Code 5: Write Single Coil
  • Function Code 6: Write Single Register
  • FC 16: Write Multiple Registers

Detection: Monitor for writes to critical registers, unexpected function codes, and addresses outside normal operating ranges.

BACnet Guide

Building Automation and Control Networks — discover devices and objects.

  • Who-Is / I-Am discovery
  • ReadProperty / WriteProperty
  • Analog Input, Binary Output, etc.
  • BACnet IP (UDP 47808/0xBAC0)

Detection: Unexpected WriteProperty commands, device re-initialization, and objects entering fault state.

OPC UA Guide

OPC Unified Architecture — platform-independent industrial communication.

  • TCP port 4840
  • Client-Server publish/subscribe
  • SecurityPolicy & authentication
  • Namespace browsing

Detection: Unauthenticated connections, anonymous binds, and unexpected node writes.

MQTT Guide

MQTT publish/subscribe messaging for IoT and SCADA telemetry.

  • QoS levels 0, 1, 2
  • Topic hierarchy (e.g., ot/plant1/plc1/temperature)
  • Wildcard subscriptions (+ and #)
  • Retained messages

Detection: Unauthorized publishers, topic injection, and unexpected retained messages.

S7 (Siemens) Guide

Siemens S7 communication — used in Siemens PLC environments.

  • TCP port 102
  • ISO-on-TCP (RFC 1006)
  • Data Blocks (DB1, DB2, ...)
  • Rack/Slot addressing

Detection: Unauthorized block writes, S7-Online tampering, and connection storms.

Detection Methodology

General approach for OT threat detection in this lab.

  1. Baseline: Learn normal traffic patterns
  2. Monitor: Use SIEM + packet capture
  3. Detect: Identify anomalies vs known attacks
  4. Classify: Fault vs cyber incident
  5. Respond: Document findings and contain

Reference the CISA OT Defense guidance.

Pro Tip: Use the Safe Practice service (port 9090) to explore each protocol without affecting the main lab environment. Start there if you're new to OT protocols!
